# Amazon Invoice 合并工具 — 项目需求文档

版本：v2.0
更新日期：2026-03-03
仓库地址：https://github.com/yinyanwen1979/amazon-invoice-merger

---

## 一、项目背景

Amazon 物流平台每周向卖家出具 Invoice（发票）文件，格式为 XLSX，每张发票包含数百至数千条运单明细。每条运单（Tracking ID）的费用拆分为多个 Charge Code 行记录，处理时需将其透视为一行，同时汇总多张发票、对账核查。人工操作耗时且容易出错，因此开发本工具实现批量自动化处理。

---

## 二、功能概述

本工具是一个 Windows 桌面程序，具备图形界面，支持批量导入多张 Amazon Invoice XLSX 文件，自动完成数据清洗、透视、合并，输出格式统一的总表 Excel 文件。

## 三、界面需求

### 3.1 整体布局

界面采用左右两栏布局。左侧为操作面板（宽约 270px），右侧为文件列表区域，底部设状态栏。

### 3.2 左侧面板

顶部显示工具名称标题栏（深蓝背景，白色文字）。标题下方显示使用说明文字，说明操作步骤。

操作按钮共四个，从上至下排列：

"添加文件"按钮（深蓝色），点击后弹出文件选择对话框，支持多选 .xlsx/.xls/.XLSX/.XLS 格式文件，已添加过的文件不重复导入。

"删除选中"按钮（深红色），删除右侧列表中已选中的文件条目。

"清空列表"按钮（深灰蓝色），清空全部已导入文件，操作前弹出确认对话框。

"开始合并"按钮（深绿色），触发合并流程，弹出保存路径选择对话框，选择路径后在后台线程中执行处理。

左侧底部展示一张吉祥物图片（mascot.png），等比例缩放至宽 230px 显示。

最底部显示版本号。

### 3.3 右侧文件列表

以表格形式展示已导入的文件，列信息包括：序号、文件名、文件路径、文件大小（自动换算 B / KB / MB）。支持多行选中。

### 3.4 状态栏

底部状态栏实时显示当前操作状态（就绪 / 正在处理某文件 / 写入中 / 完成 / 失败），右侧配有进度条，处理期间自动滚动。

## 四、数据处理需求

### 4.1 Invoice 文件格式

Amazon Invoice XLSX 文件具有以下固定结构：

第 1 行：标题（Invoice Annexure）

第 2 行：账户名、Invoice 号（G2 单元格）、不含税总额

第 3 行：账号、发票日期、税额

第 4 行：服务周期、到期日、含税总额（L4 单元格）

第 5 行：列头行，包含 Tracking ID、Charge Type、Charge Code、Tax Exclusive Charge Value (GBP) 等字段

第 6 行起：数据行

程序需自动定位第 5 行（含 "Tracking ID" 的行）作为列头，无需硬编码行号。

### 4.2 读取 Invoice 元数据

从每张发票文件读取以下元数据，用于后续处理：

Invoice 号：G2 单元格（第 2 行第 7 列），用于行排序和分组标识。

发票含税总额：L4 单元格（第 4 行第 12 列），即 Total Tax-Inclusive Charge (GBP)，作为输出总表 Q 列"发票金额（含税）"的固定值。该值为整张发票的含税总金额，同一张发票的所有行 Q 列均填写此值。

### 4.3 行过滤规则

跳过 Charge Code 为空的行（即小计 / Total 汇总行）。

其余行（Charge Type 为 Shipping Charge 或 Shipping Charge Adjustment）均参与处理。

### 4.4 分组规则

以（Invoice 号，Tracking ID）为复合主键进行分组，即相同 Tracking ID 若出现在不同发票文件中，在输出总表中作为独立行保留，不合并。同一发票内同一 Tracking ID 的多个费用行透视为一行。

### 4.5 费用列分配规则

Charge Type 为 Shipping Charge 的行，按 Charge Code 分配到对应输出列（H~N）：

| Charge Code | 输出列 |
|---|---|
| Base charge | H |
| High Cube Surcharge | I |
| Delivery Area Surcharge | J |
| Misdeclaration Handling Charge - Dimensions | K |
| Base Rate Adjustment | L |
| Non-Conveyable Surcharge | M |
| Additional Handling Fees: Girth | N |

Charge Type 为 Shipping Charge Adjustment 的行，按以下规则处理：若该行的 Charge Code 能在上表中匹配到对应列，则将 Tax Exclusive Charge Value 累加到该列（例如 Charge Code = "Misdeclaration Handling Charge - Dimensions" 则进入 K 列，"Base Rate Adjustment" 则进入 L 列）；若 Charge Code 无法匹配到上表任意列，则累加到 O 列"Shipping Charge Adjustment-运费调整"。

同一 (Invoice号, Tracking ID) 分组内，同一输出列若出现多行，数值累加。

### 4.6 合计列计算

P 列"合计（不含税）"= SUM(H:O)，即 H 至 O 列所有费用列之和，程序自动计算，精确到小数点后 2 位。

Q 列"发票金额（含税）"= 该行所属发票文件的 L4 单元格值（整张发票含税总额），同一文件所有行填写相同值，不按行计算。

### 4.7 排序规则

输出总表按 Invoice 号升序排列。同一发票内的行顺序保持读取顺序。

## 五、输出文件格式

### 5.1 列结构

| 列 | 字段名 | 说明 |
|---|---|---|
| A | Tracking ID | 运单号 |
| B | To Postcode | 目的地邮编 |
| C | Reference | 参考单号 |
| D | Billable weight (kg) | 计费重量，四位小数 |
| E | Length (cm) | 长度 |
| F | Width (cm) | 宽度 |
| G | Height (cm) | 高度 |
| H | Base charge（基础运费） | Shipping Charge / Base charge |
| I | High Cube Surcharge（体积附加费） | Shipping Charge / High Cube Surcharge |
| J | Delivery Area Surcharge（偏远地区） | Shipping Charge / Delivery Area Surcharge |
| K | Misdeclaration Handling Charge - Dimensions（错误申报费） | Shipping Charge 或 Adjustment 中同名 Charge Code |
| L | Base Rate Adjustment（费率调整） | Shipping Charge 或 Adjustment 中同名 Charge Code |
| M | Non-Conveyable Surcharge（不可传送附加费） | Shipping Charge / Non-Conveyable Surcharge |
| N | Additional Handling Fees: Girth（附加处理费：围长） | Shipping Charge / Additional Handling Fees: Girth |
| O | Shipping Charge Adjustment-运费调整 | Shipping Charge Adjustment 中无法匹配 H~N 的费用 |
| P | 合计（不含税） | SUM(H:O)，程序计算 |
| Q | 发票金额（含税） | 来自发票文件 L4 单元格，整张发票统一值 |
| R | 备注 | 留空，供用户手动填写 |

### 5.2 样式规范

表头（第 1 行）：深蓝填充（#4F8EF7），白色加粗文字，居中对齐，自动换行，行高 36。

数据行：斑马纹间隔（偶数行浅蓝 #F0F4FF）。

O 列（Shipping Charge Adjustment）：偶数行浅紫色填充（#EDE7F6）。

P 列（合计不含税）：橙黄填充，加粗深棕色文字。

Q 列（发票金额含税）：浅绿填充，加粗深绿色文字。

数值列统一使用千分位两位小数格式（#,##0.00），计费重量使用四位小数格式（#,##0.0000）。

冻结首行，开启自动筛选。自动调整列宽，最大宽度 38。

## 六、打包与部署

### 6.1 运行环境

程序面向 Windows 10 / 11 用户，打包为单文件 .exe，无需目标电脑安装 Python 或任何依赖库。

### 6.2 依赖库

| 库 | 版本要求 | 用途 |
|---|---|---|
| pandas | >=2.0.0 | 数据处理（备用） |
| openpyxl | >=3.1.0 | Excel 读写 |
| Pillow | >=10.0.0 | 界面图片加载 |
| pyinstaller | >=6.0.0 | 打包工具 |

### 6.3 资源文件

mascot.png 为界面吉祥物图片，与 main.py 同目录存放，打包时通过 --add-data 一并打入 .exe。

### 6.4 打包方式

方式一（推荐）：推送代码到 GitHub 仓库，GitHub Actions 自动在 Windows 环境执行 PyInstaller 打包，约 5 分钟完成，在 Actions 页面 Artifacts 区域下载 Amazon-Invoice-Merger-Windows.zip，解压得到 .exe。

方式二：在 Windows 电脑安装 Python 后双击 build.bat，约 3 分钟完成，生成 dist\Amazon-Invoice-Merger.exe。

## 七、已知约束

同名 Tracking ID 跨发票不合并，若需合并请在导出后手动处理。

Q 列"发票金额（含税）"为整张发票的含税总金额（L4 值），并非按行单独计算的含税额。

程序仅支持标准 Amazon Invoice 格式（表头行含 Tracking ID 字段），非标准格式文件会报错提示。
